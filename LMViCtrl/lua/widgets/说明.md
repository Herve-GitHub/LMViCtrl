以button.lua说明


**整体定位**

button.lua 定义了一个可被 QtLvglDesigner 识别、拖拽、配置、预览并最终在 LVGL 运行环境中创建的 Button 控件。

它主要分成两大部分：

1. `Button.__widget_meta`：面向设计器的控件说明书。
2. `Button.new(parent, state)`：面向运行时的控件工厂和行为实现。

换句话说，前半部分告诉设计器“这个控件有哪些属性、事件、图标、默认尺寸、绑定能力”；后半部分告诉运行时“如何真正创建 LVGL 按钮、如何应用属性、如何响应事件、如何销毁”。

**一、模块引入与对象定义**

文件开头引入两个依赖：

```lua
local lv     = require("lvgl")
local common = require("common.common")

local Button = {}
```

`lv` 是 LVGL Lua 绑定，用于创建真实的按钮、标签、设置样式和事件。  
`common` 提供通用工具，比如颜色字符串转换为 LVGL 可用的数值。

`Button` 是当前模块导出的控件对象，最后通过 `return Button` 返回，供设计器运行时 `require` 加载。

对应位置：button.lua

**二、控件元数据 `__widget_meta`**

`Button.__widget_meta` 是这个文件最核心的声明部分。Qt 端会解析这个 Lua 表，并转换成 C++ 里的 `WidgetMeta` 结构，供工具箱、属性面板、事件面板、绑定面板和预览绘制使用。

C++ 端对应结构在 WidgetMeta.h，解析逻辑在 luawidgetparser.cpp。

**1. 控件标识信息**

```lua
id             = "lv_button_basic",
name           = "Button",
description    = "示例按钮，包含 label 与尺寸/位置/样式属性",
category       = "基础对象",
icon           = "img/widgets/button.png",
preview_image  = "img/widgets/button.png",
schema_version = "1.1",
version        = "1.0",
author         = "designer",
tags           = { "button", "基础", "交互" },
```

这一段用于告诉设计器：

- 这个控件的唯一 ID 是 `lv_button_basic`。
- 在工具箱里显示名称为 `Button`。
- 归类到“基础对象”。
- 使用指定图标和预览图。
- 当前元数据协议版本是 `1.1`。
- 标签用于搜索、分类或后续扩展。

工具箱会从 Lua 文件中加载这些元数据，并按 `category` 分组显示，相关入口在 widgettoolbox.h。

对应位置：button.lua

**2. 渲染策略**

```lua
type        = "lv_button",
render_mode = "offscreen",
```

`type` 表示真实 LVGL 控件类型是 `lv_button`。  
`render_mode` 表示 Qt 设计器端如何绘制预览。

当前值是 `builtin`，含义是：设计器端使用内置按钮渲染逻辑，而不是只显示图片或完全自定义绘制。

文件里也预留了三种模式说明：
- `offscreen` ： Qt 端利用lvgl实现离屏渲染，渲染单个控件。
- `builtin`：Qt 用内置渲染器按 LVGL 类型绘制。
- `custom`：Qt 解析 `draw_hints` 或自定义绘制逻辑。
- `image`：Qt 直接使用 `preview_image` 作为占位图。

对应位置：button.lua

**3. 尺寸约束**

```lua
default_size = { w = 120, h = 44 },
min_size     = { w = 20,  h = 20 },
max_size     = { w = 4096, h = 4096 },
resizable    = true,
rotatable    = false,
```

这部分定义按钮在设计器里的默认尺寸和编辑约束：

- 默认宽高是 `120 x 44`。
- 最小尺寸是 `20 x 20`。
- 最大尺寸是 `4096 x 4096`。
- 支持缩放。
- 不支持旋转。

这些数据会影响画布上的拖拽、缩放和属性面板中的尺寸编辑。

对应位置：button.lua

**4. API 契约**

```lua
api = {
  has_draw             = true,
  has_on               = true,
  has_get_property     = true,
  has_set_property     = true,
  has_get_properties   = true,
  has_apply_properties = true,
  has_to_state         = true,
  has_destroy          = true,
},
```

这是一个“能力声明”，告诉设计器或运行时：这个控件实现了哪些标准接口。

这些接口包括：

- `draw`：创建并绘制 LVGL 对象。
- `on`：注册事件回调。
- `get_property` / `set_property`：单个属性读写。
- `get_properties` / `apply_properties`：批量属性读写。
- `to_state`：导出当前状态。
- `destroy`：销毁控件并释放资源。

对应位置：button.lua

**三、属性系统**

`properties` 是这个按钮暴露给属性面板的配置项。每个属性都包含名称、类型、默认值、显示标签、单位、范围、分组等信息。

属性面板会读取这些定义，然后自动生成不同类型的编辑器。C++ 端会按 `group` 分组显示，并把 `advanced = true` 的属性放入“高级”折叠区，相关逻辑在 propertypaneldock.cpp。

对应位置：button.lua

**1. 布局属性**

```lua
x, y, width, height
```

这四个属性控制按钮在父容器中的位置和尺寸：

- `x`：横坐标。
- `y`：纵坐标。
- `width`：宽度。
- `height`：高度。

它们都有数值范围和单位 `px`，适合在属性面板中用数字输入框编辑。

**2. 外观属性**

主要包括：

- `label`：按钮文本，默认 `OK`。
- `color`：文本颜色。
- `bg_color`：背景色。
- `font_size`：字体大小。
- `font_family`：字体名称，高级属性。
- `alignment`：文本对齐方式，枚举类型。
- `radius`：圆角。
- `border_color`：边框颜色，高级属性。
- `border_width`：边框宽度，高级属性。

其中 `alignment` 使用枚举选项：

```lua
left / center / right
```

设计器可以据此生成下拉选择框。

**3. 状态属性**

```lua
pressed_bg_color
disabled_bg_color
```

这两个属性用于描述不同状态下的视觉样式：

- `pressed_bg_color`：按下状态背景色。
- `disabled_bg_color`：禁用状态背景色。

当前运行时实现中，`disabled_bg_color` 已用于禁用状态的背景显示；`pressed_bg_color` 主要作为元数据和绘制提示存在，可供设计器预览或后续运行时扩展。

**4. 行为属性**

```lua
enabled
visible
```

- `enabled` 控制按钮是否响应事件。禁用时视觉上变灰，事件回调也不会执行。
- `visible` 控制按钮是否隐藏。

这两个属性都标记了 `bindable = true`，表示可以和外部变量绑定。

**四、数据绑定声明**

```lua
bindings = {
  { name = "label",   target = "text",    direction = "in"  },
  { name = "enabled", target = "enabled", direction = "in"  },
  { name = "visible", target = "visible", direction = "in"  },
  { name = "clicked", target = "trigger", direction = "out" },
},
```

这部分说明按钮支持哪些数据绑定能力：

- `label` 可以由外部变量输入，用于更新文本。
- `enabled` 可以由外部变量输入，用于启用或禁用。
- `visible` 可以由外部变量输入，用于显示或隐藏。
- `clicked` 是输出事件，可作为触发信号。

属性面板会把绑定信息作为只读概览展示，逻辑在 propertypaneldock.cpp。

对应位置：button.lua

**五、事件定义与事件处理代码**

事件定义部分：

```lua
events = {
  clicked,
  single_clicked,
  double_clicked
}
```

它声明按钮支持三类交互事件：

- `clicked`：按钮点击。
- `single_clicked`：单击。
- `double_clicked`：双击。

对应位置：button.lua

事件代码属性部分：

```lua
event_properties = {
  on_clicked_handler,
  on_single_clicked_handler,
  on_double_clicked_handler
}
```

这部分不是普通样式属性，而是让用户在设计器中填写 Lua 事件处理代码。每一项都指定：

- 对应事件名。
- 代码语言是 Lua。
- 是否多行。
- 编辑器默认行数。
- 默认代码片段或说明。

运行时会从实例属性中取出这些代码字符串，编译成 Lua 函数，再通过控件的 `on` 方法绑定到真实事件上。对应运行时逻辑在 runtime.lua。

对应位置：button.lua

**六、绘制提示 `draw_hints`**

```lua
draw_hints = {
  shape              = "rounded_rect",
  fill_from          = "bg_color",
  text_from          = "label",
  text_color_from    = "color",
  font_size_from     = "font_size",
  ...
}
```

`draw_hints` 是给 Qt 设计器端使用的通用绘制提示。它把“怎么画”与“从哪个属性取值”连接起来。

例如：

- 按钮形状来自 `shape = "rounded_rect"`。
- 填充色来自 `bg_color`。
- 文本来自 `label`。
- 文本颜色来自 `color`。
- 字号来自 `font_size`。
- 圆角来自 `radius`。
- 边框来自 `border_color` 和 `border_width`。

这使得设计器无需真正运行完整 LVGL，也能根据属性在画布上画出接近真实效果的按钮预览。C++ 端绘制提示入口在 drawHints.h。

对应位置：button.lua

**七、运行时工厂 `Button.new(parent, state)`**

`Button.new(parent, state)` 是控件实例的创建入口。运行时会根据工程里的 `WidgetInstance` 构造 `state`，再调用控件模块的 `new` 方法创建真实 LVGL 对象。对应调用逻辑在 runtime.lua。

对应位置：button.lua

它的执行流程可以概括为：

1. 接收父对象 `parent` 和初始状态 `state`。
2. 根据 `properties` 的默认值初始化 `self.props`。
3. 定义一组内部样式应用函数。
4. 创建 LVGL 按钮对象和标签对象。
5. 应用初始尺寸、位置、文本和样式。
6. 提供事件注册接口。
7. 提供属性读写接口。
8. 提供状态导出和销毁接口。

**八、属性初始化**

```lua
self.props = {}
for _, p in ipairs(Button.__widget_meta.properties) do
  self.props[p.name] = state[p.name] ~= nil and state[p.name] or p.default
end
```

这段逻辑保证每个属性都有值：

- 如果外部 `state` 传入了属性，就使用外部值。
- 如果没有传入，就使用元数据中的默认值。

这样按钮实例从创建开始就是完整状态，不会出现缺少属性导致绘制异常的问题。

对应位置：button.lua

**九、样式应用函数**

文件中定义了一组局部函数，用于集中处理样式：

- `apply_bg_color`：应用背景色。
- `apply_text_color`：应用文本颜色。
- `apply_font_size`：应用字体大小。
- `apply_alignment`：应用文本对齐。
- `apply_radius`：应用圆角。
- `apply_border`：应用边框。
- `apply_visible`：应用显示/隐藏。
- `apply_enabled_visual`：根据启用状态应用正常或禁用背景色。

这些函数的好处是：初始化和后续属性变更都复用同一套逻辑，避免样式处理分散在多个地方。

对应位置：button.lua

**十、真实 LVGL 对象创建**

```lua
self.btn = lv.btn_create(parent)
self.btn:set_size(self.props.width, self.props.height)
self.btn:set_pos(self.props.x, self.props.y)

self.label = lv.label_create(self.btn)
self.label:set_text(self.props.label)
self.label:center()
```

这里创建了两个 LVGL 对象：

- `self.btn`：按钮主体。
- `self.label`：按钮内部文本。

之后依次应用背景、文本颜色、字体、对齐、圆角、边框和可见性。

对应位置：button.lua

**十一、事件订阅机制**

`self:on(event_name, callback)` 提供统一事件注册接口。

它支持：

- `clicked`
- `single_clicked`
- `double_clicked`

内部会把字符串事件名转换为 LVGL 事件常量，例如 `lv.EVENT_CLICKED`。回调执行时使用 `pcall` 包裹，避免用户事件代码报错后影响整个运行时。

还有一个重要逻辑：

```lua
if not self.props.enabled then return end
```

也就是说，按钮被禁用后，即使 LVGL 事件触发，业务回调也不会执行。

对应位置：button.lua

**十二、属性读写机制**

`get_property(name)` 返回单个属性。  
`set_property(name, value)` 修改属性，并根据属性名立即更新真实 LVGL 对象。

例如：

- 修改 `label` 会调用 `self.label:set_text(value)`。
- 修改 `color` 会刷新文本颜色。
- 修改 `bg_color` 或 `disabled_bg_color` 会刷新按钮背景。
- 修改 `x` 或 `y` 会重新设置位置。
- 修改 `width` 或 `height` 会重新设置尺寸。
- 修改 `enabled` 会刷新启用/禁用视觉。
- 修改 `visible` 会显示或隐藏控件。

这使属性面板中的变更可以实时反映到运行时对象或预览对象上。

对应位置：button.lua

**十三、批量属性、状态导出与销毁**

```lua
get_properties()
apply_properties(props_table)
to_state()
destroy()
```

这几个方法用于工程保存、加载、批量更新和资源释放：

- `get_properties`：复制并返回当前属性表。
- `apply_properties`：批量调用 `set_property`。
- `to_state`：导出当前状态，便于保存。
- `destroy`：删除 LVGL 对象，清空按钮和标签引用。

销毁时会优先调用对象自身的 `del` 方法，如果不可用则回退到 `lv.obj_del`，兼容不同 LVGL Lua 绑定风格。

对应位置：button.lua

**十四、给客户汇报时可以这样总结**

这个 button.lua 采用“元数据声明 + 运行时实现”的双层结构：

- 元数据层负责描述控件身份、分类、图标、尺寸、属性、事件、绑定和绘制提示。
- 设计器通过解析元数据，自动生成工具箱项、属性面板、事件编辑区、绑定说明和画布预览。
- 运行时层负责真正创建 LVGL 按钮对象，管理文本、颜色、尺寸、可见性、启用状态和事件回调。
- 属性系统是统一入口，既能支持设计器实时编辑，也能支持工程保存加载。
- 事件系统支持用户在设计器里编写 Lua 代码，并在运行时动态绑定到按钮点击、单击和双击事件。
- 整体结构具有良好的扩展性，后续新增控件时可以沿用同样模式，只需要补充对应的元数据和运行时行为。

一句话汇报版：

“这个按钮脚本不是简单的控件示例，而是一个完整的设计器控件插件单元。它通过 `__widget_meta` 向 Qt 设计器声明可视化编辑能力，通过 `new()` 在 LVGL 运行时创建真实控件，从而实现从工具箱展示、属性编辑、事件配置、数据绑定到最终运行的完整闭环。”
