图形化控件事件绑定系统 — 原型设计文档
版本：v1.0 · 2026年5月12日

一、设计目标
用户可以通过拖拽连线的方式，将控件的事件（Event）与目标控件的动作（Action）或数据变量绑定，无需手写代码，实现直观的交互设计。

核心原则：

零代码优先 — 常见场景（显示/隐藏、文本赋值、数值联动）完全拖拽完成
渐进式复杂度 — 简单场景一步完成，高级场景（条件、序列、脚本）在详情面板扩展
可逆操作 — 所有连线操作支持 Ctrl+Z 撤销
类型安全提示 — 锚点拖拽时实时灰显不兼容目标，防止无效绑定

七、一对多绑定 — 动作序列模型
7.1 核心设计
每个事件锚点对应一个有序动作队列，而非简单分叉线。画布上收束为一根主线加编号徽章，编辑时展开完整序列面板。

  画布收起状态（浏览时整洁）:

  ┌─ ■ Button1 ──────────────────────┐
  │                                   │
  │            onClick ●━━━(3)━━━━▶  │   ← 小徽章显示动作数
  │         onLongPress ●             │     悬停 tooltip 预览列表
  └───────────────────────────────────┘

  画布展开状态（点击徽章后）:

  ┌─ ■ Button1 ─────────┐
  │      onClick ●━━━━━━━┿━━━━━━━┬──────────────────────────┐
  └─────────────────────┘        │  动作序列                 │
                                  │  ① ━━▶ Label1.setText    │
                                  │  ② ━━▶ $count.increment  │
                                  │  ③ ━━▶ Image1.show       │
                                  │  [+ 拖入新目标]          │
                                  └──────────────────────────┘
7.2 动作序列执行流程
  顺序模式:

  ⚡ Button1.onClick 触发
       │
       ▼
  ① Label1.setText       参数: "已点击 " + $count   条件: 无      延迟: 0ms
       │
       ▼
  ② $count.increment     参数: +1                   条件: 无      延迟: 0ms
       │
       ▼
  ③ Image1.show          参数: —                    条件: $count>3  延迟: 200ms


  并行模式:

  ⚡ 并行触发
       ├──▶ ① Label1.setText
       ├──▶ ② $count.increment
       └──▶ ③ Image1.show
7.3 动作序列能力对比
能力	原分叉方案	动作序列方案
执行顺序控制	❌	✅ 拖动编号调序
单条动作独立禁用	❌	✅
单条动作独立条件	❌	✅
单条动作独立延迟	❌	✅
顺序 / 并行切换	❌	✅
画布可读性	线条杂乱	✅ 收束为徽章

九、触摸屏事件规范
触摸屏环境不应使用双击事件，原因：

响应延迟 — 系统等待第二次点击判断，导致单击有约 300ms 延迟
操作不可靠 — 手指间隔难以精确控制，误触率高
无统一语义 — 触摸屏 UX 规范中双击无约定俗成含义
  触摸屏标准事件集:

  ✅ onClick          单点触发
  ✅ onLongPress      长按（替代双击，承担次要操作语义）
  ✅ onPress          按下瞬间
  ✅ onRelease        手指抬起
  ✅ onSwipeLeft      向左滑动
  ✅ onSwipeRight     向右滑动
  ✅ onSwipeUp        向上滑动
  ✅ onSwipeDown      向下滑动
  ❌ onDoubleClick    移除，改用 onLongPress
